# 视觉技能融合说明

本次加入 4 个运行时技能：视觉路由、完成封面/诊断、16:9 小黑正文配图/shot list、材质解释图/图表美化。它们与已有运营技能通过 Visual Brief 交接，而非把视觉方法塞进排期、选题或文案技能。

| 来源 | 可使用的融合方式 | 许可/边界 |
| --- | --- | --- |
| `ponyodong2026/ponyo-cover-anchor-system` | 信息密度 × 视觉锚点、四类模板、封面诊断、生活方式变体 | 未发现可复用许可；不复制提示词、模板或成图。 |
| `helloianneo/ian-xiaohei-illustrations` | 16:9 小黑正文配图、shot list、风格 DNA、构图与 QA | MIT；保留归属，不复制示例成图或构图。 |
| `op7418/guizang-material-illustration` | 材质解释图、图表语义重画、参考只补事实、中心图与外层卡片分工 | 未发现可复用许可；不复制素材包、提示词或模板。 |

`cangjie-skill` 只用于这次融合的蒸馏与测试流程，不是这四个技能的运行时依赖、方法或用户入口。

## Skill 更新摘要

### wb-xhs-visual-router

新增：

- 共用 Visual Brief，把选题、初稿、排期中的标题、证据、页面和视觉约束接入视觉生产。
- 封面标题、来源、主体与事实边界的最小预检。
- 完成封面/诊断、16:9 小黑正文配图/shot list、材质解释图/图表美化三类请求的明确分流。
- `called` / `not_called` 的真实运行状态、保存路径和 QA 记录。

核心变化：视觉交付不再散落在运营技能中；所有视觉请求先经过真实预检和分流，未调用工具时不会虚构图片已完成。

### wb-xhs-cover-anchor

新增：

- 从 `ponyodong2026/ponyo-cover-anchor-system` 转译的信息密度 × 视觉锚点、冲突/数字/截图/情绪模板、封面诊断与生活方式变体。
- 3:4 画幅、80px 缩略图、完成封面提示词、手机端标题可读性和事实中性兜底。

核心变化：封面优先依托已确认事实，同时可以诊断和改版旧封面；无可用证据时保留标题与构图骨架，不复刻来源的提示词、模板或成图。

### wb-xhs-xiaohei-illustration

新增：

- 从 `helloianneo/ian-xiaohei-illustrations` 吸收的 16:9 白底正文配图、4-8 张 shot list、小黑动作、红橙蓝批注、构图与 QA。
- 当前文章的认知锚点和原创低科技隐喻，而非来源示例的构图。
- 用户明确选择铅笔、人物叙事、3:4 和顶部标题留白时的 `pencil_narrative` 变体；它独立于小黑默认的来源能力，不把默认画幅或角色职责混掉。

核心变化：把文章里的判断、流程、状态和隐喻变成可读的正文认知配图，同时保留 MIT 来源归属，不复制示例成图或构图。来源方法保护可交付能力，当前主题的构图、动作、物件和隐喻保留创造空间。

### wb-xhs-material-illustration

新增：

- 从 `op7418/guizang-material-illustration` 转译的材质解释图、图表语义重画、参考只补事实与中心图/外层卡片分工。
- 3-5 个短标签、箭头、图例、数据与关系，以及从中心图拆出的物件、步骤、关系、角标和说明组件。
- “素材 + 封面”请求中的标题优先预检与双交付拆分。

核心变化：先让中心图讲清关系或数据，再沉淀可复用组件；不复制来源素材包、模板或未授权提示词。
