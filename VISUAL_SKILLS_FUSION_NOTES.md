# 视觉技能融合说明

本次加入 4 个运行时技能：视觉路由、封面锚点、黑白叙事插图和模块化素材插画。它们与已有运营技能通过 Visual Brief 交接，而非把视觉方法塞进排期、选题或文案技能。

| 来源 | 可使用的融合方式 | 许可/边界 |
| --- | --- | --- |
| `ponyodong2026/ponyo-cover-anchor-system` | 转译为证据、主体、对照、文字四类封面锚点 | 未发现可复用许可；不复制提示词、模板或成图。 |
| `helloianneo/ian-xiaohei-illustrations` | 简洁黑白叙事插图方向 | MIT；保留归属，不复制具体角色与成图。 |
| `op7418/guizang-material-illustration` | 模块化素材组织思路 | 未发现可复用许可；不复制素材包、提示词或模板。 |

`cangjie-skill` 只用于这次融合的蒸馏与测试流程，不是这四个技能的运行时依赖、方法或用户入口。

## Skill 更新摘要

### wb-xhs-visual-router

新增：

- 共用 Visual Brief，把选题、初稿、排期中的标题、证据、页面和视觉约束接入视觉生产。
- 封面标题、来源、主体与事实边界的最小预检。
- 封面、黑白叙事插图、模块化素材三类请求的明确分流。
- `called` / `not_called` 的真实运行状态、保存路径和 QA 记录。

核心变化：视觉交付不再散落在运营技能中；所有视觉请求先经过真实预检和分流，未调用工具时不会虚构图片已完成。

### wb-xhs-cover-anchor

新增：

- 从 `ponyodong2026/ponyo-cover-anchor-system` 转译的证据、主体、对照、文字承诺四类封面锚点。
- 3:4 画幅、手机端标题可读性和事实中性兜底。

核心变化：封面优先依托已确认事实；无可用证据时保留标题与构图骨架，不复刻来源的提示词、模板或成图。

### wb-xhs-xiaohei-illustration

新增：

- 从 `helloianneo/ian-xiaohei-illustrations` 吸收的简洁黑白叙事方向。
- 人物动作、情绪关系、标题留白和单页用途的输出结构。

核心变化：把抽象主题变成可读的黑白叙事插图，同时保留 MIT 来源归属，不复制具体角色、成图或真实人物形象。

### wb-xhs-material-illustration

新增：

- 从 `op7418/guizang-material-illustration` 转译的模块化素材组织思路。
- 物件、步骤、关系、角标、说明组件以及统一视觉语言。
- “素材 + 封面”请求中的标题优先预检与双交付拆分。

核心变化：栏目素材可沉淀为可复用组件，而不是一次性拼图；不复制来源素材包、模板或未授权提示词。
